This post is the synthesis of an argument I have been circling for a few essays now.

React felt powerful because it moved us one layer up.

Instead of manually mutating the DOM, we declared what the UI should be and let the framework handle the imperative machinery underneath.

I think AI engineering is going through the same kind of abstraction shift.

But the important artifact is not just "the prompt."

It is the declaration around the task:

- prompts or notes that transmit intent
- specs and scenarios that structure intent
- validation loops that compare the running product against that intent

> Prompts start the interaction. Specs stabilize the intent. Validation closes the loop.

That is why I care less about clever prompt wording and more about the strength of the intent layer.

The implementation still matters. But more of the leverage is moving into the layer above it: behavior, constraints, scenarios, and proof.

This is also why validation matters so much. Without it, the declaration is just aspiration. With it, the agent can reconcile reality against the intended system.

That is the real analogy to React for me: not prompts replacing code, but engineering becoming more declarative because intent is getting a stronger layer above implementation.
