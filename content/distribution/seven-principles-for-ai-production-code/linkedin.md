If I had to compress my AI coding workflow into a few durable rules, it would be these seven:

1. Rich context beats clever wording.
2. Understanding comes before implementation.
3. Scenarios are the contract.
4. Feedback loops matter more than models.
5. Write the shape before the code.
6. Review behavior first, style second.
7. Concurrency is a reward for discipline.

What I like about this list is that it is not tied to any one tool.

It is really describing an operating system around the model.

That system matters more than most people think.

A strong model with weak feedback still drifts.
A decent model with strong feedback often converges.

And if you zoom out, all seven are pointing at the same pattern:

- make intent explicit
- make feedback fast
- make review disciplined

That is the layer that turns AI from "interesting demo" into something I can trust near production work.
